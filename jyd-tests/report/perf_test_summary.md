# 三个样本的行为性能测试汇总报告

## 1. 总体说明

`src0.bin`、`src1.bin`、`src2.bin` 的性能测试入口都为 `run_perf_test_stage()`，三者整体测试框架高度一致，不是单纯追求吞吐的裸 benchmark，而是“带正确性门禁的性能测试”。

三个样本共有的测试行为如下：

1. 先读取输入值，并显示到七段数码管。
2. 通过向 `MMIO_COUNTER` 连续写入 10 次启动值来启动硬件计数器。
3. 串行执行 3 组工作负载。
4. 每组工作负载都带有结果校验；只要任意一步失败，就立即停止后续测试。
5. 结束时停止计数器，读取耗时，并把耗时追加显示到七段数码管。
6. 最后通过 LED 点阵输出通过或失败图案。

因此，这 3 个样本的共同特点是：

- 都把“结果正确”放在“性能计时”之前；
- 都采用“失败立即退出”的控制方式；
- 都通过 MMIO 外设对测试结果进行可视化反馈。

## 2. 三个样本各自的测试行为

### 2.1 `src0`：矩阵、排序、素数统计的交叉校验型测试

`src0` 的三组工作负载分别是：

1. `90 x 90` 矩阵乘法交叉校验  
   对 10 轮不同种子的矩阵进行初始化，分别用两套矩阵处理路径生成结果，再逐项比较输出是否一致。

2. 1000 元素排序交叉校验  
   对同一组输入数据复制出两份，分别执行冒泡排序和选择排序，再比较排序结果是否完全一致。

3. 素数统计交叉校验  
   分别用试除法和筛法统计 `2..20000` 范围内的素数数量，最后比较两个计数结果是否相同。

`src0` 的行为特点是“同一逻辑结果由两种不同实现或路径来互相校验”。  
它覆盖了矩阵计算、排序、数值算法三类典型负载，测试重点是：

- 算术与访存路径是否稳定；
- 不同实现之间是否能得到一致结果；
- 在较长执行链上是否出现中途失配。

### 2.2 `src1`：状态机、矩阵、排序的串行一致性测试

`src1` 的三组工作负载分别是：

1. 状态序列一致性校验  
   维护一个 4 状态状态机，根据输入值和模运算规则推进状态，把状态写入缓冲区，并对两轮长循环的加权和与固定常量进行比较。

2. `100 x 100` 矩阵乘法重复校验  
   初始化两张矩阵后，连续两次执行相同的矩阵乘法流程，并比较两份输出矩阵是否完全一致。

3. 3000 元素排序交叉校验  
   对同一输入数组复制出两份，分别执行插入排序和选择排序，再比较排序结果。

`src1` 的行为特点是“在规则驱动的长时状态演化基础上，再叠加计算与排序校验”。  
相比 `src0`，它的第一组负载更偏向控制流和状态转移验证，测试重点是：

- 分支路径和状态推进是否稳定；
- 长循环中的状态记录与累加结果是否可重复；
- 大规模计算和排序流程在后续阶段是否仍能保持一致性。

### 2.3 `src2`：链表、CRC、矩阵的结构一致性测试

`src2` 的三组工作负载分别是：

1. 链表构造与加权求和校验  
   针对大量不同 `seed` 构造长度固定为 2000 的链表，比较相同种子链表的加权和、不同种子链表的差异，并验证链表长度和逆序复制后的求和结果。

2. CRC 翻转一致性校验  
   对 4KB 缓冲区计算 CRC，随后翻转一个字节再次计算，再恢复该字节并第三次计算，要求“翻转后 CRC 改变，恢复后 CRC 复原”。

3. `90 x 90` 矩阵乘法重复校验  
   初始化两张矩阵后，使用“右矩阵先转置再乘法”的实现连续计算两次，并比较结果矩阵是否一致。

`src2` 的行为特点是“面向数据结构与数据完整性的一致性测试”。  
相比前两个样本，它更强调：

- 动态结构构造后的内容一致性；
- 数据扰动是否能被 CRC 检测；
- 数据恢复后校验值是否可复现；
- 矩阵计算在重复执行时是否保持确定性。

## 3. 三个样本的差异总结

如果把三者放在一起看，可以概括为：

- `src0` 偏向“算法交叉验证”，重点覆盖矩阵、排序、素数统计。
- `src1` 偏向“状态控制 + 计算验证”，重点覆盖状态机长流程、矩阵和排序。
- `src2` 偏向“数据结构 + 完整性验证”，重点覆盖链表、CRC 和矩阵。

三者虽然具体负载不同，但测试理念一致：  
先证明结果可信，再把整段执行时间作为性能测试结果输出。

## 4. 结论

这 3 个样本都不是脱离正确性的单纯性能跑分程序，而是将“行为验证”和“性能计时”绑定在一起的综合测试阶段。

从测试设计上看：

- `src0` 更适合观察多种经典算法路径的一致性；
- `src1` 更适合观察复杂控制流和长序列状态行为的稳定性；
- `src2` 更适合观察链式结构、校验算法和重复计算的确定性。

因此，若把它们作为一组样本来看，可以认为它们分别从：

- 算法结果一致性，
- 状态迁移一致性，
- 数据结构与数据完整性一致性

三个角度，对系统的行为性能进行了覆盖式测试。
