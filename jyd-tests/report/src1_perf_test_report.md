# src1 性能测试报告

## 概述

`src1.bin` 的性能测试入口为 `run_perf_test_stage()`。
这一阶段的总体流程是：

- 在七段数码管上显示输入值
- 启动硬件计数器
- 依次执行三个带正确性校验的工作负载
- 任一工作负载失败时立即停止并上报失败
- 全部通过后停止计数器、显示耗时，并输出通过图案

与 `src0` 一样，这一阶段不是“只测性能不管结果”的裸 benchmark，而是一个带正确性门禁的性能测试。

## 入口与显示行为

性能阶段开始时会先调用：

- `show_input_on_seg()`
- `counter_start_pulse10()`

`show_input_on_seg()` 会读取 `0x80100000` 处的输入值，将其转换为压缩十进制数字后写入 `MMIO_SEG`。

`counter_start_pulse10()` 会向 `MMIO_COUNTER` 连续写入 10 次 `0x80000000`，用于启动计时。

## 工作负载 1：状态序列一致性校验

第一部分调用 `state_sequence_test_stage()`。
这一步不是传统矩阵或排序工作负载，而是一个长时间运行的状态转移测试：

- 维护一个 4 状态的状态机
- 按索引生成输入值并驱动状态转移
- 将每一步的状态写入一段 256 字节缓冲区
- 对两轮大循环分别累加加权和
- 将两轮结果都与固定常量 `0x0f7d39a9` 比较

其中状态转移由 `advance_state_mod_pattern()` 决定，核心规则包括：

- 在状态 `0` 下按奇偶切换
- 在状态 `1` 下检查对 `3` 的模
- 在状态 `2` 下检查对 `5` 的模
- 在状态 `3` 下检查对 `7` 的模

只有当两轮累加结果都满足预期时，这一阶段才视为通过。

## 工作负载 2：矩阵乘法交叉校验

第二部分使用两个 `100 x 100` 矩阵进行交叉校验：

- `fill_mod_12343_matrix_100x100(0x80105ef0, 1)` 初始化矩阵 A
- `fill_mod_12343_matrix_100x100(0x8010fb30, 2)` 初始化矩阵 B
- `matrix_mul_100x100()` 计算第一份结果到 `0x80119770`
- 再次调用 `matrix_mul_100x100()` 计算第二份结果到 `0x801233b0`
- `matrix_eq_100x100()` 比较两份输出

矩阵填充逻辑比较简单：

- `out[row * 100 + col] = (row + col + seed) mod 50`

矩阵乘法本身是标准三重循环实现。
这部分的作用更像是对大规模整数乘加路径和内存访问路径做稳定性校验；只有两份结果完全一致才继续后续测试。

## 工作负载 3：排序交叉校验

第三部分会执行 10 轮排序校验，`iter = 0..9`。
每轮流程如下：

- `fill_linear_expr_matrix_100x100(0x80100130, 3000, iter)` 生成 3000 个输入元素
- 将该缓冲区逐元素复制到 `0x80103010`
- `insertion_sort_u32(0x80100130, 3000)` 对第一份数据做插入排序
- `selection_sort_u32(0x80103010, 3000)` 对第二份数据做选择排序
- `array_eq_u32()` 比较两份排序结果

输入数据的生成公式为：

- `value = (count * 0x49 - seed * 0x1f + index) mod 0x3037`

因此这部分不是在比较两组不同数据，而是在同一组输入上交叉验证两种排序实现。
只要任意一轮比较失败，性能阶段就会立刻进入失败上报路径。

## 通过/失败上报

失败路径调用 `report_perf_fail()`，成功路径调用 `report_perf_pass()`。
两者的共同模式是：

- `counter_stop_pulse10()`
- `read_counter()`
- `or_decimal_digits_to_seg(value)`

区别在于最后输出到 LED 的图案不同：

- 失败时调用 `build_led_fail_pattern()`
- 成功时调用 `build_led_pass_pattern()`

因此，`src1` 的性能测试结果通过两种 MMIO 侧效果体现：

- 七段数码管显示计时值
- LED 点阵显示通过/失败图案

## 结论

`src1` 的性能测试可以概括为三个串行的正确性保护工作负载：

- 状态序列与加权和校验
- `100 x 100` 矩阵乘法结果校验
- 3000 元素排序结果校验

它的特点不是工作负载种类特别多，而是每一阶段都先验证结果是否可信，再决定是否继续计时流程。
也就是说，`src1` 的“性能测试”本质上是“带结果门禁的性能测试”，而不是脱离正确性的纯吞吐测试。
